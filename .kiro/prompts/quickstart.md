# Quickstart: Project Setup Wizard

## Welcome to the Kiro Hackathon!

This interactive wizard will configure your project for the Dynamous Kiro Hackathon.

## Setup Process

### 1. Project Information
- **Project Name**: What's your project called?
- **Description**: Brief description of what it does
- **Problem Solved**: What real-world problem does it address?
- **Target Users**: Who will use this application?

### 2. Technical Stack
- **Primary Language**: Python, JavaScript, etc.
- **Frameworks**: FastMCP, Flask, React, etc.
- **Database**: Snowflake, PostgreSQL, etc.
- **APIs/Services**: External services you'll integrate

### 3. Kiro Configuration
- **Custom Prompts**: Which workflow prompts do you need?
- **Steering Documents**: Update product.md, tech.md, structure.md
- **Development Workflow**: How will you use Kiro CLI?

### 4. Documentation Setup
- **README Template**: Generate project-specific README
- **DEVLOG Structure**: Set up development log format
- **Architecture Docs**: Create technical documentation outline

## Configuration Files Updated

After running this wizard:
- ✅ `.kiro/steering/` documents customized
- ✅ `.kiro/prompts/` workflow configured  
- ✅ `README.md` template generated
- ✅ `DEVLOG.md` structure created
- ✅ Development workflow established

## Next Steps

1. **Load context**: Run `@prime` to understand your setup
2. **Plan features**: Use `@plan-feature` for your first feature
3. **Start building**: Execute plans with `@execute`
4. **Maintain quality**: Regular `@code-review` sessions

## Tips for Success

- **Document everything**: Your process is 20% of the score
- **Use Kiro extensively**: CLI usage is 20% of the score
- **Build something useful**: Real-world value is heavily weighted
- **Stay organized**: Clean structure impresses judges
